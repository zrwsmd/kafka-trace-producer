# Kafka Trace 数据传输 - 三端部署完整指南

> **架构**: 上位机(Producer) → 云服务器(Kafka Broker) → 控制端(Consumer)

---

## 一、架构总览

```
┌──────────────────────────────────────────────────────────────────────┐
│                        完整数据流                                      │
└──────────────────────────────────────────────────────────────────────┘

  ① 上位机 (ARM64)                ② 云服务器 (AMD64)              ③ 控制端 (PC/Web)
  ┌─────────────────┐            ┌─────────────────┐            ┌─────────────────┐
  │ Java 程序        │            │ Kafka Broker    │            │ Node.js 程序     │
  │ (Producer)      │            │ (消息中间件)     │            │ (Consumer)      │
  │                 │            │                 │            │                 │
  │ TraceSimulator  │   发送数据   │ Topic:          │   拉取数据   │ 读取并显示       │
  │ ↓               │ ────────→  │ trace-data      │ ←──────── │ 轨迹数据         │
  │ KafkaProducer   │   TCP 9092 │ (持久化存储)     │   TCP 9092 │                 │
  │                 │            │                 │            │                 │
  │ JAR 包          │            │ Kafka 3.6.1     │            │ kafkajs         │
  └─────────────────┘            └─────────────────┘            └─────────────────┘
  
  部署位置: 现场设备              部署位置: 阿里云ECS            部署位置: 办公室/家里
  运行方式: java -jar xxx.jar    运行方式: systemd 后台服务     运行方式: node consumer.js
```

---

## 二、三端角色说明

| 端 | 角色 | 部署位置 | 运行内容 | 职责 |
|---|------|---------|---------|------|
| **① 上位机** | **Kafka Producer** | ARM64 设备 (现场) | `kafka-trace-producer-1.0.0.jar` | 生产数据，发送到 Kafka |
| **② 云服务器** | **Kafka Broker** | 阿里云 ECS (AMD64) | Kafka 服务 (后台运行) | 接收、存储、转发消息 |
| **③ 控制端** | **Kafka Consumer** | 你的 PC / Web 服务器 | Node.js 消费者程序 | 消费数据，显示/分析 |

---

## 三、部署步骤 (按顺序执行)

### 【第一步】云服务器: 部署 Kafka Broker

> **在阿里云 ECS (AMD64, Ubuntu 20.04/22.04) 上执行**

#### 3.1 上传部署脚本

```bash
# 在你的本地电脑 (Windows)
cd kafka-trace-producer
scp -r deploy/ root@你的云服务器IP:/root/
```

#### 3.2 执行部署

```bash
# SSH 登录云服务器
ssh root@你的云服务器IP

# 进入部署目录
cd /root/deploy/

# ① 安装 Kafka (下载 + 解压 + 配置用户)
bash install-kafka.sh

# ② 配置 Kafka (自动检测公网IP, 配置监听地址)
bash configure-kafka.sh

# ③ 启动 Kafka + 创建 trace-data topic
bash start-kafka.sh

# ④ 验证 Kafka 是否正常
bash test-kafka.sh
```

#### 3.3 开放防火墙端口

```bash
# 阿里云控制台 → 安全组 → 添加入站规则
# 端口: 9092
# 协议: TCP
# 授权对象: 0.0.0.0/0 (或仅允许上位机IP)
```

#### 3.4 设为开机自启 (可选)

```bash
cd /root/deploy/
bash setup-systemd.sh

# 以后可以用 systemd 管理
sudo systemctl status kafka
sudo systemctl restart kafka
sudo journalctl -u kafka -f  # 查看日志
```

**✅ 云服务器部署完成! 记下公网IP, 如: `123.45.67.89`**

---

### 【第二步】上位机: 部署 Kafka Producer (Java JAR)

> **在你的开发电脑上编译，然后上传到 ARM64 上位机**

#### 3.5 编译 JAR 包 (在开发电脑 Windows 上)

```bash
# 前提: 已安装 Maven
cd kafka-trace-producer

# 修改配置文件
notepad application.properties
# 将 kafka.bootstrap.servers=YOUR_SERVER_IP:9092
# 改为 kafka.bootstrap.servers=123.45.67.89:9092 (你的云服务器公网IP)

# 编译打包
mvn clean package -DskipTests

# 生成的 JAR 包位置:
# target/kafka-trace-producer-1.0.0.jar
```

#### 3.6 上传到上位机

```bash
# 方式一: SCP 上传
scp target/kafka-trace-producer-1.0.0.jar root@上位机IP:/opt/trace/
scp application.properties root@上位机IP:/opt/trace/

# 方式二: U盘拷贝 (如果上位机无网络)
# 将 JAR 包和 application.properties 拷贝到 U盘
# 插入上位机, 复制到 /opt/trace/
```

#### 3.7 在上位机运行

```bash
# SSH 登录上位机 (ARM64)
ssh root@上位机IP

# 确认已安装 Java (JRE 8+)
java -version
# 如果没有: sudo apt install openjdk-11-jre-headless

# 运行 Producer
cd /opt/trace/
java -jar kafka-trace-producer-1.0.0.jar application.properties

# 正常输出:
# === Kafka Trace Producer (1000万帧零丢失) ===
# === Trace Config ===
#   kafka.bootstrap.servers = 123.45.67.89:9092
#   trace.max.frames        = 10000000
# [KafkaProducer] initialized
# [TraceSimulator] starting...
# [TraceSimulator] progress: 1.0% (1000/100000 batches, 100000 frames)
```

#### 3.8 后台运行 (可选)

```bash
# 使用 nohup 后台运行
nohup java -jar kafka-trace-producer-1.0.0.jar application.properties \
    > trace.log 2>&1 &

# 查看日志
tail -f trace.log

# 查看进程
ps aux | grep kafka-trace-producer

# 停止
pkill -f kafka-trace-producer
```

**✅ 上位机开始发送数据!**

### 【第三步】控制端: 部署 Kafka Consumer (Node.js)

> **在你的办公室 PC / Web 服务器上运行**

#### 3.9 创建 Node.js Consumer 项目

```bash
# 在你的开发电脑 (Windows)
mkdir trace-consumer
cd trace-consumer
npm init -y
npm install kafkajs
```

#### 3.10 创建消费者代码

创建 `consumer.js`:

```javascript
const { Kafka } = require('kafkajs');

// ===== 配置 =====
const KAFKA_BROKER = '123.45.67.89:9092';  // 改为你的云服务器公网IP
const TOPIC = 'trace-data';
const GROUP_ID = 'trace-consumer-group';

// ===== 创建 Kafka 客户端 =====
const kafka = new Kafka({
  clientId: 'trace-consumer',
  brokers: [KAFKA_BROKER],
  retry: {
    retries: 5,
    initialRetryTime: 300,
  },
});

const consumer = kafka.consumer({ groupId: GROUP_ID });

// ===== 统计 =====
let batchCount = 0;
let frameCount = 0;
const startTime = Date.now();

// ===== 主函数 =====
async function run() {
  console.log('=== Kafka Trace Consumer (Node.js) ===');
  console.log(`  Broker: ${KAFKA_BROKER}`);
  console.log(`  Topic:  ${TOPIC}`);
  console.log('');

  // 连接
  await consumer.connect();
  console.log('✓ Connected to Kafka');

  // 订阅 topic (从头消费)
  await consumer.subscribe({ topic: TOPIC, fromBeginning: true });
  console.log(`✓ Subscribed to topic: ${TOPIC}`);
  console.log('');
  console.log('开始消费数据... (Ctrl+C 退出)\n');

  // 消费消息
  await consumer.run({
    eachMessage: async ({ topic, partition, message }) => {
      batchCount++;

      // 解析 JSON
      const payload = JSON.parse(message.value.toString());
      const seq = payload.seq;
      const frames = payload.frames || [];
      frameCount += frames.length;

      // 每 100 包打印一次进度
      if (batchCount % 100 === 0) {
        const elapsed = (Date.now() - startTime) / 1000;
        const speed = batchCount / elapsed;
        console.log(`[Consumer] 已消费: ${batchCount} 包, ${frameCount} 帧  |  ` +
                    `速度: ${speed.toFixed(0)} 包/秒  |  ` +
                    `offset: ${message.offset}  |  ` +
                    `最新seq: ${seq}`);
      }

      // ===== 在这里处理数据 =====
      // 例如: 存入数据库、实时显示、分析等
      // const firstFrame = frames[0];
      // console.log('第一帧数据:', firstFrame);
    },
  });
}

// ===== 优雅退出 =====
process.on('SIGINT', async () => {
  console.log('\n\n=== 消费统计 ===');
  console.log(`总包数: ${batchCount}`);
  console.log(`总帧数: ${frameCount}`);
  const elapsed = (Date.now() - startTime) / 1000;
  console.log(`耗时: ${elapsed.toFixed(1)} 秒`);
  console.log(`平均速度: ${(batchCount / elapsed).toFixed(0)} 包/秒`);
  await consumer.disconnect();
  process.exit(0);
});

// ===== 启动 =====
run().catch(console.error);
```

#### 3.11 运行 Consumer

```bash
node consumer.js
```

**正常输出:**

## 五、常见问题

### Q1: 上位机连接不上 Kafka

**现象**: `Connection refused` 或 `Timeout`

**排查**:
```bash
# ① 检查云服务器 Kafka 是否运行
ssh root@云服务器IP
ps aux | grep kafka

# ② 检查端口是否监听
netstat -tuln | grep 9092

# ③ 检查安全组是否开放 9092
# 阿里云控制台 → 安全组 → 入站规则

# ④ 测试连通性 (在上位机上)
telnet 云服务器IP 9092
```

### Q2: 控制端收不到数据

**排查**:
```bash
# ① 检查 topic 是否有数据
ssh root@云服务器IP
/opt/kafka/bin/kafka-console-consumer.sh \
  --bootstrap-server localhost:9092 \
  --topic trace-data \
  --from-beginning \
  --max-messages 1

# ② 检查 Consumer Group 状态
/opt/kafka/bin/kafka-consumer-groups.sh \
  --bootstrap-server localhost:9092 \
  --describe \
  --group trace-consumer-group
```

### Q3: 如何查看 Kafka 日志

```bash
# Kafka Broker 日志
tail -f /opt/kafka/logs/server.log

# 如果用了 systemd
sudo journalctl -u kafka -f
```

### Q4: 如何重置消费进度 (重新消费)

```javascript
// 在 consumer.js 中修改:
await consumer.subscribe({ 
  topic: TOPIC, 
  fromBeginning: true  // 从头消费
});

// 或者删除 Consumer Group
// ssh root@云服务器IP
// /opt/kafka/bin/kafka-consumer-groups.sh \
//   --bootstrap-server localhost:9092 \
//   --delete --group trace-consumer-group
```

## 六、性能调优

### 上位机 (Producer)

编辑 `application.properties`:
```properties
# 加快发送速度 (减小间隔)
trace.send.interval.ms=10   # 默认 50, 改为 10 → 速度提升 5 倍

# 或全速发送
trace.send.interval.ms=0    # 不限速, 几分钟发完 1000 万帧
```

### 控制端 (Consumer)

```javascript
// 增加批量拉取数量
await consumer.run({
  eachBatch: async ({ batch }) => {
    // 批量处理, 性能更高
    for (let message of batch.messages) {
      // ...
    }
  },
});
```
